Setup & Usage
① RTK · Rust Token Killer

⚡ CLI Output Compression (60–90%)

Intercepts shell command output and compresses it before it reaches your LLM context. Single Rust binary, zero dependencies.

Install
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh
# or macOS homebrew
brew install rtk
Init for Cursor
rtk init -g --agent cursor
# Restart Cursor after this
Common Usage
rtk git status        # ~2000 → ~200 tokens
rtk cargo test        # strips test pass noise
rtk ls .              # token-optimized tree
rtk read file.rs      # smart file reading
rtk read file.rs -l aggressive  # sigs only
rtk grep "pattern" . # grouped results
rtk gain              # see your savings
rtk discover          # find missed savings
  • Hook auto-rewrites bash commands — no manual prefix needed after init
  • Errors & failures are always preserved — only noise is stripped
  • Windows: extract zip, add to PATH, use rtk init -g --agent cursor in Cursor rule
② code-parser · M97chahboun

🔬 Structural Code Analysis

Fast Rust CLI that statically analyses source files and returns a structured JSON map of every class, interface, mixin, enum, and method with exact line ranges — without executing code.

Install
# Via cargo
cargo install --git https://github.com/M97chahboun/code-parser

# Or download binary from releases
# github.com/M97chahboun/code-parser/releases
Usage
# Parse a single file
code-parser src/main.rs

# Parse entire directory
code-parser src/

# Output to file for Cursor context
code-parser src/ > .cursor/codebase-map.json
Use in Cursor
# Reference the map instead of raw files
# In chat: "@.cursor/codebase-map.json"
# Sends structure, not full file bodies
  • Returns line ranges per symbol — LLM can ask for only the methods it needs
  • No runtime execution — safe for any codebase, any language
  • Regenerate the map after major refactors: code-parser src/ > .cursor/codebase-map.json
  • Supports Dart, Rust, Python, TypeScript and more
③ Byterover · brv CLI

🧠 Persistent Memory Layer

Gives AI agents persistent, structured memory across sessions. Curate project knowledge into a context tree so Cursor doesn't re-learn your codebase every session.

Install
npm install -g byterover-cli
# or
pip install byterover
Connect to Cursor
brv init               # start interactive REPL
brv webui              # open dashboard
brv status             # check project status
# Byterover auto-picks connector type for Cursor
# (Skill connector — no config needed)
Curate Knowledge
brv curate "Auth uses JWT with 24h expiry" @src/middleware/auth.ts
brv curate "DB is PostgreSQL, ORM is Drizzle"
brv query  "How is auth implemented?"
brv curate view        # see stored knowledge
  • Curate arch decisions, patterns, and constraints once — retrieved automatically
  • Works locally by default — cloud sync available for teams
  • Free tier available at byterover.dev
④ Modular .mdc Rules · Built-in

📐 Scoped Context Rules (Free)

Replace a single monolithic .cursorrules with modular .mdc files that only load when relevant. Each always-apply rule costs tokens on every prompt — scope them.

Folder Structure
.cursor/rules/
├── core.mdc         # always-on, keep <300 lines
├── testing.mdc      # globs: **/*.test.ts
├── api.mdc          # globs: src/routes/**
├── ui.mdc           # globs: src/components/**
└── security.mdc     # globs: **/*.auth.*
Example .mdc frontmatter
.cursor/rules/testing.mdc
--- description: Testing conventions globs: - "**/*.test.ts" - "**/*.spec.ts" alwaysApply: false --- # Testing Rules - Use Vitest, not Jest - Colocate tests next to source - Never mock DB calls in E2E tests - Test command: npm run test:e2e
  • Keep core.mdc tiny — it loads on every prompt
  • Use alwaysApply: false + globs to trigger conditionally
  • Commit .cursor/rules/ to Git — team shares the same AI behavior
  • Enable in Cursor: Settings → Rules → Project Rules
⑤ Caveman · JuliusBrussee

🪨 Response Compression (~75%)

A Cursor/Claude Code skill that instructs the AI to respond in compressed caveman-style language — dropping filler words and verbose phrasing while keeping full technical accuracy.

Install (Cursor)
# Copy caveman.mdc into your rules folder
curl -o .cursor/rules/caveman.mdc \
  https://github.com/JuliusBrussee/caveman/raw/main/caveman.skill

# Or clone and copy manually
git clone https://github.com/JuliusBrussee/caveman
cp caveman/caveman.skill .cursor/rules/caveman.mdc
How It Works
# Normal AI response:
"I will now proceed to refactor the authentication
 middleware to use JWT tokens as you requested."

# Caveman response (same meaning, 75% fewer tokens):
"Refactor auth middleware → JWT. Done."
Activate Per Session
# Reference it in your prompt or as always-apply rule
# .cursor/rules/caveman.mdc frontmatter:
alwaysApply: true   # on for every response
alwaysApply: false  # manual: @caveman in prompt
  • Works on AI responses — complements rtk which compresses tool outputs
  • 928 GitHub stars — widely validated in production
  • Zero dependencies — a single .mdc text file, nothing to install
  • Disable for final code reviews where full explanations matter
Workflow Best Practices
1 🔄
Context Hygiene
Start Fresh for Every Task
Always open a new agent session for each new task or user story. Never build on top of a long context from a previous task.
  • One session = one task or user story
  • Long contexts cause hallucination & forgotten instructions
  • Use byterover to carry knowledge between sessions instead
  • Cmd+N → New Chat in Cursor
2 📊
Context Budget
Keep Context at 50–70% Max
Monitor the context length indicator in Cursor. Once you hit 70%, quality degrades fast — reset before that happens.
✓ 0–70% Safe zone ⚠ 70–85% ✗ 85%+ Reset now
  • Best results live in the 50–70% range
  • Above 70%: model starts forgetting early instructions
  • Curate key decisions to byterover before resetting
3 ⚡
Power Prompting
Pre-craft Prompts with AI Chat Tools
Before sending a prompt to Cursor agent, use a chat tool to draft a compact, precise prompt. Don't waste agent tokens on vague instructions.
  • Describe your task in detail to ChatGPT, Groq, or Qwen
  • Ask it: "Write a compact Cursor agent prompt for this"
  • Copy the result into Cursor — shorter, cleaner, less context wasted
  • Groq is fastest for quick prompt rewrites (free tier)
4 🗺️
Plan Before Agent
Use Plan Mode First, Then Agent
Always run Plan Mode to review the implementation approach before switching to Agent Mode. A bad plan wastes 5–15 model calls to undo.
  • Toggle Plan Mode in the Cursor chat panel
  • Review the plan — correct scope, files, and approach
  • Only switch to Agent Mode once the plan is approved
  • A single plan exchange costs far less than a failed agent run
Quick Reference
RTK Top Commands
  • rtk gain — savings stats
  • rtk discover — find missed savings
  • rtk smart file.rs — 2-line summary
  • rtk diff f1 f2 — condensed diff
  • rtk proxy <cmd> — raw passthrough
code-parser Tips
  • Use JSON map as codebase index
  • Reference map in .cursorrules
  • Regenerate after big refactors
  • Combine with @file for precision
  • Avoids full file body in context
Byterover Tips
  • Curate arch decisions immediately
  • Add tech stack & DB choices
  • Curate recurring error patterns
  • Use brv query to verify retrieval
  • Review pending with brv review pending
MDC Rule Types
  • Always — every prompt (use sparingly)
  • Auto — when files match glob
  • Agent-requested — LLM pulls on demand
  • Manual — you attach explicitly
  • Mix types to minimize baseline cost
Caveman Tips
  • Enable globally in core.mdc for max savings
  • Disable for onboarding / documentation tasks
  • Stack with rtk for output + response compression
  • Single .mdc file — no binary needed
  • Works with Cursor, Claude Code, Windsurf
